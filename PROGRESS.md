## 已确认待开发事项

- 目前只支持“单条自定义模型配置 + 显式启用 / 停用”，还不支持多条自定义模型列表
- `openai_compatible` 当前只提供单条配置和基础参数校验，还没有更多高级请求参数 UI
- 半屏阅读时，PDF 对照面板的空间分配还可以继续优化
- 停止翻译后的状态提示和页面反馈还可以更顺手
- 合并回归时仍能看到既有 SQLite `ResourceWarning`，需要后续单独清理
- 2026-03-30：翻页性能优化已立项，基准文档为 `2011 - Michel Foucault.pdf`；已确认瓶颈主要在 `/reading` 单请求内重复读取和重复处理页面数据，而不是 SQLite 查询本身
- 当前决策：本轮先做请求内去重和上下文复用，不先做浏览器回归，不上进程级 LRU，不直接进入 AJAX 翻页
- 第一阶段目标：把 `/reading` 服务端耗时从约 `1136ms/1637ms` 压到 `200–400ms` 区间；若仍高于 `600ms` 或降幅不足 `30%`，进入第二阶段“轻量页索引接口拆分”
- 第二阶段预案：把“整页正文内容”与“翻页/进度元数据”拆成两套读取接口，让状态链路不再依赖 1.5MB 全量 markdown 反序列化
- 2026-03-30：第一阶段已完成；当前同口径复测中，`/reading` 已译页约 `158ms`、待译页约 `151ms`、顺序翻页前 20 页约 `157ms`，单请求内 `load_pages_from_disk/load_entries_from_disk` 已降到各 `1` 次，暂不进入第二阶段
