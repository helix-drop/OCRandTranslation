# 外文文献阅读器

一个在本机运行的外文文献阅读工具：

- 上传 PDF 或图片
- 调用 PaddleOCR 解析版面
- 用 Qwen / DeepSeek 翻译
- 支持单条自定义模型配置并显式启用
- 在阅读页对照查看 PDF 原文、译文和脚注

适合“边读边翻、边核对原文”的使用方式，不需要部署服务器，也不需要数据库配置。

## 先准备什么

开始前只需要准备两样：

1. Python 3.10 或更高版本
2. 可用的 API Key
   - `PaddleOCR 令牌`
   - `Qwen（DashScope）API Key` 或 `DeepSeek API Key`

## 新手最快上手

### 第 1 步：启动程序

macOS / Linux：

```bash
./start_managed.sh
```

Windows 10 / 11：

```powershell
.\start_managed.ps1
```

如果 PowerShell 不方便，也可以直接双击：

```bat
start_managed.bat
```

这些启动脚本会自动完成下面几件事：

1. 首次创建 `.venv` 虚拟环境
2. 安装或更新依赖
3. 启动程序
4. 尝试打开浏览器到 `http://localhost:8080`

### 当前唯一推荐启动方式

- `start_managed.sh` / `start_managed.ps1` / `start_managed.bat`
  - 会用专用 Chrome / Edge 窗口打开应用
  - 关闭这个专用窗口后，应用进程会自动结束
  - 如果系统里没有可用的 Chrome / Edge，脚本会提示并退出

旧的 `start.sh` / `start.ps1` / `start.bat` 已移除，不再维护。

### 第 2 步：填 API Key

打开浏览器后：

1. 进入“设置”
2. 填入 `PaddleOCR 令牌`
3. 再填入一个翻译模型的 Key
   - 要么填 `DashScope API Key`
   - 要么填 `DeepSeek API Key`
4. 保存设置

如果你想用自定义模型，也是在“设置”页里完成：

- 先在“翻译模型”区域展开 `自定义模型`
- 填 `Provider`、`模型 ID`，按需填写 `Base URL`
- 先点“保存自定义模型配置”
- 再点“启用此自定义模型”

当前支持三类自定义模型：

- `Qwen`：复用 `DashScope API Key`
- `DeepSeek`：复用 `DeepSeek API Key`
- `OpenAI Compatible`：需要单独填写 `Base URL` 和专用 `API Key`

如果你已经开始正式翻译，可以再看一眼“翻译性能”里的 `段内并发翻译`：

- 默认关闭
- 开启后，单页会同时翻译多个段落，通常更快
- 这个功能会增加 API 并发请求数
- 如果你遇到限流、超时或不稳定，先把它关掉
- 并发上限由你自己设置，范围是 `1-10`
- 数值越高，请求越密集，也越容易触发限流、超时或单页失败

### 第 3 步：上传 PDF 开始读

回到首页后：

1. 上传 PDF 或图片
2. 等待 OCR 解析完成
3. 点击“从 p.1 开始读”或“继续读”
4. 在阅读页里按需要开启 PDF 原文对照

## 如果你想手动启动

### macOS / Linux

```bash
python3 -m venv .venv
source .venv/bin/activate
python3 -m pip install -r requirements.txt
python3 app.py
```

### Windows PowerShell

```powershell
py -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
python app.py
```

如果你的系统没有 `py`，也可以把第一行改成：

```powershell
python -m venv .venv
```

## 你会看到什么

程序默认运行在：

```text
http://localhost:8080
```

首页主要有三件事：

1. 上传文档
2. 进入设置填写 Key
3. 继续当前文档阅读

阅读页主要有这些能力：

- 逐页阅读译文
- 查看原文和脚注
- 开启左侧 PDF 对照
- 拖拽 PDF 面板宽度
- 从指定页开始翻译
- 导出 Markdown（标题层级自动与目录结构对齐）
- 导出时可选过滤非主体页（版权页、重复封面、部分书目前置页）

设置页里还可以按需开启 `段内并发翻译`。这个开关默认是关闭的，适合先以稳定为主上手；确认你的模型额度和网络都稳定后，再自己把并发上限逐步调高。

设置页现在也会显示：

- 当前生效的翻译模型 ID
- 当前目录索引文件（如已导入）
- 当前目录文件的导入时间
- 直接重新上传替换目录文件的入口

## 术语词典

你可以为每份文档维护一份术语词典，翻译时会自动命中，重译时同样生效。

### 批量导入（首页）

在首页的文档列表里，每张文档卡片的操作区有「词典与目录」按钮，点击后弹出窗口，切换到术语词典块，支持：

- 选择 `.xlsx` 或 `.csv` 文件一次性导入
- **追加模式**：与现有词典合并，同名术语以新值覆盖
- **覆盖模式**：清空现有词典后全量写入

**文件格式**

| 列 | 内容 |
|---|---|
| 第一列 | 源语言术语（如法语原文） |
| 第二列 | 中文译文 |

- 首行如果包含"术语""term""译文""translation"等表头字样，会自动跳过
- 直接从数据行开始也可以
- 其余列忽略，空行自动跳过

CSV 示例：

```
raison,理性
monomanie,偏执狂
Esquirol,艾斯基洛尔
```

### 逐条管理（设置页）

进入设置页，滚动到"术语词典"区域，可以逐条添加、编辑和删除，与批量导入互不干扰，共用同一份词典。

## 书籍目录导航

适合书籍类 PDF，上传目录后阅读页工具栏会出现「目录」下拉，可按章节跳转。论文、单篇文章无需使用此功能。

### 自动提取目录

上传 PDF 后，系统会按以下顺序自动尝试提取目录，无需额外操作：

1. **PDF 书签**：读取 PDF 文件内置的 outline（书签），大多数正式出版的电子书都有这一层
2. **目录页超链接**（fallback）：若 PDF 没有书签，系统会扫描文档前 30% 的页面，找到内嵌了大量内部跳转链接的目录页，从中提取章节标题与页码

自动提取的目录层级（depth）会同步影响导出 Markdown 中标题的 `#` 层数（见下方导出说明）。

### 手动导入目录（首页）

自动提取结果不理想时，可在首页文档卡片的「词典与目录」按钮中手动导入，上传三列 `.xlsx` 或 `.csv` 文件：

| 列 | 内容 | 示例 |
|---|---|---|
| 第一列 | 章节标题 | `第一章 引言` |
| 第二列 | 层级深度（整数） | `0`（章）、`1`（节）、`2`（小节） |
| 第三列 | 原书印刷页码 | `15` |

CSV 示例：

```
第一章 引言,0,1
1.1 研究背景,1,3
1.2 研究方法,1,7
第二章 文献综述,0,12
```

首行含"标题""title""页码""page"等表头字样时自动跳过，空行自动忽略。上传的原始文件会持久化保存，重新上传新文件时自动替换。

### 当前目录文件状态

导入目录后，你可以在两个地方看到“当前正在使用的目录索引文件”：

- **首页「词典与目录」弹层**：显示当前目录文件名、导入时间、目录条数和页码偏移
- **设置页**：显示同样的信息，并支持直接重新上传替换目录文件

目录原文件会一直保留，直到被新的目录文件替换；不会无提示地消失。

### 页码偏移校准

书籍 PDF 通常有封面、版权页等无页码的前置页，导致原书印刷页码与 PDF 文件页序不一致。上传目录后系统会尝试自动校准，也可以手动指定：

> **偏移含义**：原书第 1 印刷页对应 PDF 的第几页（1-based）。例如封面占 3 页，则填 4。

校准入口有两处，效果相同：

- **首页「词典与目录」**：导入成功后弹出校准框，确认或修改后保存
- **阅读页「目录」下拉底部**：随时调整，保存后立即生效，无需刷新页面
- **设置页**：已导入目录时会显示「书籍目录页码偏移」卡片，可随时修改

### 目录导航（阅读页）

工具栏出现「目录」按钮后：

- 点击展开下拉列表，显示各章节标题与原书页码
- 点击任意条目即跳转到对应页
- 当前所在章节自动高亮

## 导出 Markdown

在阅读页工具栏点击「导出」后可下载整本书的 Markdown 文件，格式约定如下：

- **原文**：用 `> ` 引用块包裹
- **译文**：紧跟原文后的普通段落
- **脚注（默认）**：优先导出为 Obsidian 标准脚注 `[^label]` / `[^label]: ...`，定义就近放在对应段落后
- **尾注（高置信）**：仅在高置信判定时分流到 `## 本章尾注` 或 `## 全书尾注`
- **回退块**：无法可靠解析编号时，保留为 `[脚注] ...` / `[脚注翻译] ...` 普通文本块，避免伪造编号
- **标题**：`#` 数量由目录层级（depth）决定
  - depth=0（章） → `#`
  - depth=1（节） → `##`
  - depth=2（小节） → `###`
  - 无目录匹配时退化为 OCR 识别的字体大小层级

### 按章节选择导出

在导出弹窗中，书名下方会列出所有顶级章节及其页码范围。默认全选（导出整本书）；取消勾选某些章节后，只导出选中章节的页面内容。未提取到目录时不显示章节列表，直接导出全书。

### 省略非主体页

导出弹窗中可勾选「省略非主体页（版权/广告/重复封面）」：

- 默认不勾选，保持与历史导出一致
- 勾选后会过滤前置版式页，并去掉前几页中高相似的重复封面内容
- 与章节选择导出可叠加使用

## 数据放在哪里

所有用户数据都保存在项目目录下的 `local_data/`，不会写到系统级目录：

- `local_data/user_data/config.json`
  保存 API Key、模型偏好、自定义模型配置、术语表和部分阅读设置
- `local_data/user_data/data/app.db`
  SQLite 主库，保存文档、页面、翻译状态、目录文件元数据和结果
- `local_data/user_data/data/documents/{doc_id}/source.pdf`
  每份文档的 PDF 副本
- `local_data/user_data/data/documents/{doc_id}/toc_source.xlsx` 或 `.csv`
  当前生效的目录原始文件（如有）；会一直保留到下一次替换

这些内容默认不会提交到 Git 仓库。

## 常见问题

### 1. 浏览器没有自动打开

手动访问：

```text
http://localhost:8080
```

### 2. 页面提示缺少 Key

去“设置”页先保存：

- `PaddleOCR 令牌`
- 一个可用的翻译模型 Key

### 3. 第一次启动比较慢

这是正常现象。首次启动会创建虚拟环境并安装依赖，通常会比后续启动慢很多。

### 4. 上传后一直没结果

先检查：

1. `PaddleOCR 令牌` 是否正确
2. 当前网络是否能访问外部 API
3. PDF 是否损坏或过大

### 5. 想停止程序

在启动程序的那个终端窗口里按 `Ctrl + C` 即可。
